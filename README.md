# Enterprise Knowledge Assistant

An AI-powered **Retrieval-Augmented Generation (RAG)** system for enterprise document question answering. The application enables users to upload PDF documents, perform intelligent semantic and keyword-based search, and receive grounded responses generated by a local Large Language Model (LLM) with source citations.

The system combines **Hybrid Retrieval (BM25 + Vector Search)**, **Reciprocal Rank Fusion (RRF)**, **CrossEncoder Re-ranking**, **Conversational Memory**, and **Ollama (Llama 3.2)** to improve retrieval accuracy while minimizing hallucinations.

---

## Features

* Hybrid Retrieval (BM25 + Vector Search)
* Reciprocal Rank Fusion (RRF)
* CrossEncoder Re-ranking
* Conversational Memory
* Incremental PDF Upload & Indexing
* ChromaDB Vector Database
* Source Citations
* Confidence Score
* Enterprise Chat Interface
* Dark / Light Theme
* Evaluation Framework
* Local LLM Inference using Ollama

# Installation

## 1. Clone the Repository

```bash
git clone https://github.com/<your-username>/enterprise-knowledge-assistant.git

cd enterprise-knowledge-assistant
```

---

## 2. Create a Virtual Environment

### Windows

```bash
python -m venv venv

venv\Scripts\activate
```

### Linux / macOS

```bash
python3 -m venv venv

source venv/bin/activate
```

---

## 3. Install Dependencies

```bash
pip install -r requirements.txt
```

---

## 4. Install Ollama

Download Ollama from:

https://ollama.com/download

Pull the required model:

```bash
ollama pull llama3.2
```

Verify installation:

```bash
ollama list
```

---

## 5. Add Documents

Place PDF documents inside:

```
app/data/documents/
```

---

## 6. Build the Vector Database

```bash
python -m app.rag.ingest
```

---

## 7. Run the Application

```bash
python main.py
```

Open your browser:

```
http://127.0.0.1:5000
```

---

# Running the Evaluation

Execute the evaluation framework:

```bash
python -m app.evaluation.run_evaluation
```

The evaluation reports:

* Answer Similarity
* Retrieval Precision
* Retrieval Recall
* Confidence Score
* Response Latency

Results are exported to:

```
evaluation_results.csv
```

# Current Limitations

* Supports PDF documents only.
* Conversation memory is session-based.
* Confidence score uses a heuristic approach.
* Local deployment using the Flask development server.
* Response generation is synchronous.
